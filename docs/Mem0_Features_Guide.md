# Mem0 核心功能全整理文档

## 文档说明
本文档基于 Mem0 官方功能文档及相关特性解析，梳理了 Mem0 的全量核心功能，涵盖记忆基础管理、高级检索、个性化配置、数据交互、场景化适配、集成能力六大类，同时标注各功能核心用途、关键参数及典型使用场景，方便开发与使用参考。

---

## 一、记忆基础管理功能

### 1. 上下文式记忆添加（Contextual Add）
*   **核心用途**：自动管理对话上下文，无需手动追踪历史记录，仅传入新消息即可实现记忆的关联存储，支持记忆的分层管理。
*   **关键特性**：
    *   仅传新消息，Mem0 自动关联用户历史记忆，减少请求体数据量。
    *   **支持用户维度记忆（仅 user_id）**：存储长期持久化信息（如用户偏好、过敏史），所有会话通用。
    *   **支持会话专属记忆（user_id + run_id）**：存储单任务 / 单话题上下文（如工单咨询、旅行规划），不同会话完全隔离。
*   **典型场景**：个人 AI 助手、客户服务、教育平台的分层记忆管理。

### 2. 直接导入（Direct Import）
*   **核心用途**：跳过 Mem0 的智能记忆推导环节，将预定义的标准化记忆直接存入系统，提升存储效率。
*   **关键参数**：`infer=False`（开启直接导入的核心开关，默认 `infer=True`）。
*   **关键特性**：
    *   不提炼、不加工，原封不动存储 `user` 角色消息原文，忽略 `assistant`/`system` 角色消息。
    *   跳过重复检测，同一内容多次导入会重复存储。
    *   调用 `add` 后返回空列表，无处理后的记忆对象返回。
*   **典型场景**：批量导入标准化用户档案、已知固定信息、脱敏后的核心业务数据。

### 3. 记忆时间戳（Memory Timestamps）
*   **核心用途**：为记忆指定实际事件发生时间，替代系统默认的 “录入时间”，保障记忆时间线的准确性。
*   **关键格式**：需传入 Unix 时间戳（秒级整数）（从 1970-01-01 UTC 起的秒数）。
*   **关键特性**：记忆的时间属性与实际事件一致，不受录入系统的时间影响，支持时间维度的精准排序与检索。
*   **典型场景**：补录历史记忆、迁移旧系统数据、基于时间线的用户行为分析。

### 4. 记忆过期时间（Expiration Date）
*   **核心用途**：为记忆设置有效期限，到期后记忆自动退出检索结果，实现时效性信息的精细化管理。
*   **关键格式**：传入标准日期字符串（如 `2025-12-31`），也可通过时间函数动态计算。
*   **关键特性**：记忆到期后仅不参与 `search`/`get` 等检索，原始数据仍保留在系统中，可修改过期时间恢复检索。
*   **典型场景**：存储临时行程、限时活动、短期通知、工单临时信息等时效性内容。

### 5. 记忆删除 / 批量操作
*   **核心用途**：支持单条 / 批量删除记忆，或删除用户 / 智能体相关实体，清理无效 / 过期记忆。
*   **关联工具**：MCP 集成中的 `delete_memory`（单条删除）、`delete_all_memories`（批量删除）、`delete_entities`（实体删除）。
*   **典型场景**：清理测试数据、删除过期的临时记忆、批量清理某一用户的全部记忆。

## 二、高级检索功能

### 1. 高级检索（Advanced Retrieval）
*   **核心用途**：突破基础的语义相似度检索，通过多维度增强方式实现精准的记忆查找与排序。
*   **核心增强选项**：**关键词搜索（Keyword Search）** —— 扩展结果至包含特定词汇、人名、专业术语的记忆，提升召回范围，适配领域专属术语检索。
*   **典型场景**：检索特定实体 / 专业术语、需要对某一主题做全面覆盖的场景。

### 2. 条件检索（Criteria Retrieval）
*   **核心用途**：基于自定义规则（如情绪、意图、行为信号）定义 “相关性”，对记忆进行加权排序，实现意图感知的精细化检索。
*   **关键配置**：自定义检索条件（含 `name` 名称、`description` LLM 解析说明、`weight` 权重），通过 `client.project.update(retrieval_criteria=...)` 全局生效。
*   **关键特性**：
    *   排序逻辑：语义相似度 + LLM 基于自定义条件的加权评分。
    *   支持临时关闭：检索时设置 `use_criteria=False` 即可恢复标准检索。
    *   分数分布贴合权重设置，对核心特征（如愉悦度、紧急度）精准加权。
*   **典型场景**：健康管理助手（优先愉悦记忆）、高效办公智能体（降权消极记忆）、辅导类智能体（突出好奇心记忆）。

### 3. 图谱记忆（Graph Memory）
*   **核心用途**：构建实体间的关联关系网络，在语义检索基础上补充实体关系上下文，提升检索结果的场景关联性。
*   **关键参数**：`enable_graph=True`（开启图谱记忆的核心开关）。
*   **关键特性**：
    *   Mem0 自动为实体构建 “节点 - 边” 的图谱化表达。
    *   图谱关系随语义检索结果一同返回，不改变向量检索结果的原始排序。
    *   支持搭配任意重排序器（Reranker）使用。
*   **典型场景**：需要追踪实体关系的场景（如电商商品关联、金融客户需求关联、教育知识点关联）。

### 4. 图谱存储阈值（Graph Store Threshold）
*   **核心用途**：作为图谱记忆的底层配置，控制实体匹配的严格程度，决定两个实体是否判定为 “同一个” 并合并节点。
*   **核心逻辑**：基于嵌入向量相似度的数值门槛，阈值越高匹配越严格（避免误合并），阈值越低匹配越宽松（自动合并相似实体）。
*   **典型场景**：金融 / 医疗等精准领域（高阈值，避免实体误判）、客服 / 社交类应用（低阈值，合并用户同义表述）。

## 三、个性化配置功能

### 1. 自定义指令（Custom Instructions）
*   **核心用途**：通过自然语言规则，精准定义 Mem0 从对话中提取 / 排除的信息，充当记忆提取的智能过滤器，保障记忆质量与隐私。
*   **配置方式**：通过 `client.project.update(custom_instructions=...)` 为项目全局配置，支持分类定义提取规则、排除敏感信息。
*   **高级用法**：**条件化处理** —— 根据对话类型（如客服 / 销售 / 新用户引导）设置差异化的提取规则。
*   **典型场景**：电商客服（仅提取商品问题、客户偏好）、健康应用（仅提取健康信息，排除个人标识）、金融顾问（仅提取理财需求，排除银行卡信息）。

### 2. 自定义属性 / 权重（搭配条件检索）
*   **核心用途**：为记忆提取与检索定义个性化属性，通过权重控制各属性对结果的影响程度，适配不同业务的核心诉求。
*   **典型属性**：joy（愉悦度）、negativity（消极度）、curiosity（好奇心）、urgency（紧急度）、risk（风险等级）等。
*   **关键特性**：属性描述需清晰明确，建议配置 3-5 个核心属性，避免冗余 / 模糊。

### 3. 自定义分类（Custom Categories）
*   **核心用途**：对记忆进行自定义分类管理，方便按业务维度快速筛选与检索，提升记忆的组织性。
*   **典型场景**：将用户记忆分为 “个人偏好”“业务需求”“历史行为” 等类别，按类别检索。

## 四、数据交互功能

### 1. 记忆导出（Memory Export）
*   **核心用途**：将 Mem0 中的记忆按自定义结构导出为结构化数据，适配业务的数据二次使用（如分析、迁移、报表）。
*   **关键配置**：自定义 Pydantic Schema 数据模型，定义导出数据的字段、类型、约束；支持 `export_instructions` 自定义数据处理规则。
*   **检索导出数据**：支持按导出 ID 检索、按过滤条件检索（如 `user_id`/`created_at`）。
*   **可用过滤条件**：`user_id`/`agent_id`/`run_id`/`session_id`/`created_at`（时间范围）。
*   **典型场景**：导出用户记忆做行为分析、将 Mem0 数据迁移至其他业务系统、生成记忆相关的业务报表。

### 2. 反馈机制（Feedback Mechanism）
*   **核心用途**：对 Mem0 生成的记忆提交评价反馈，Mem0 基于反馈持续学习，优化记忆生成与检索的准确性。
*   **反馈类型**：`POSITIVE`（有用）、`NEGATIVE`（无用）、`VERY_NEGATIVE`（完全无用）。
*   **关键操作**：
    *   单条反馈：`client.feedback(memory_id=..., feedback=..., feedback_reason=...)`。
    *   批量反馈：循环调用反馈接口，适配大反馈量场景。
    *   删除反馈：为 `feedback`/`feedback_reason` 传入 `None`/`null`。
*   **最佳实践**：反馈理由需具体可操作（如 “包含过期联系信息”），避免模糊表述（如 “内容错误”）。
*   **反馈分析**：支持监控反馈完成率、反馈分布、记忆质量趋势、用户满意度指标。

### 3. Webhooks（网络钩子）
*   **核心用途**：配置实时通知，当 Mem0 中发生记忆创建 / 更新 / 删除事件时，自动向指定 URL 发送 HTTP POST 请求，实现 Mem0 与业务应用的实时联动。
*   **项目级配置**：每个 Webhook 绑定特定项目，仅接收该项目的事件通知。
*   **核心操作**：创建（`create_webhook`）、查询（`get_webhooks`）、更新（`update_webhook`）、删除（`delete_webhook`）。
*   **支持事件类型**：`memory_add`（记忆新增）、`memory_update`（记忆更新）、`memory_delete`（记忆删除）。
*   **最佳实践**：实现重试逻辑、验证请求来源、异步处理事件、监控钩子运行状态。

## 五、场景化适配功能

### 1. 群聊记忆（Group Chat）
*   **核心用途**：支持多参与者对话的记忆管理，将记忆自动归属至对应发言者，为每位参与者建立独立的记忆档案。
*   **核心优势**：相较 OpenAI Memory，准确率提升 26%，延迟降低 91%，令牌消耗减少 90%。
*   **关键识别条件**：消息中包含 `name` 字段（参与者名称），Mem0 自动识别为群聊场景。
*   **记忆归属规则**：
    *   用户消息：`name` 作为 `user_id`。
    *   助手 / 智能体消息：`name` 作为 `agent_id`。
    *   无名称消息：退回到标准角色识别。
*   **关键特性**：
    *   **支持异步模式**：`async_mode=True`，提升大人数 / 大篇幅群聊的处理性能。
    *   **精准检索**：支持按会话（`run_id`）、按参与者（`user_id`）检索群聊记忆，`user_id="*"` 匹配所有参与者。
*   **典型场景**：团队会议（追踪成员贡献）、多客户客服（隔离不同客户记忆）、多智能体系统（管理智能体间对话）、协作项目讨论。

### 2. 多角色记忆管理（User/Agent/System）
*   **核心用途**：区分不同角色的消息来源，实现用户、AI 助手、系统的记忆独立存储与检索，避免记忆混淆。
*   **关键角色**：`user`（人类参与者）、`assistant`（AI 助手）、`agent`（智能体）、`system`（系统提示）。
*   **典型场景**：多智能体协作场景，区分不同智能体的记忆与用户记忆。

## 六、集成与扩展功能

### 1. MCP 集成（Model Context Protocol Integration）
*   **核心用途**：通过模型上下文协议（MCP）实现任意 AI 客户端与 Mem0 的无缝连接，无需手动调用 API，让 AI 智能体自主管理记忆。
*   **核心优势**：通用兼容性（支持 Claude Desktop、Cursor、自定义智能体）、智能体自主化（自主决定记忆增删改查）、零基础设施成本、协议标准化。
*   **开放工具**：共 9 个核心记忆操作工具，覆盖全生命周期管理：
    *   `add_memory` / `search_memories` / `get_memories`
    *   `update_memory` / `delete_memory` / `delete_all_memories` / `delete_entities`
    *   `get_memory` / `list_entities`
*   **部署方式**：Python 包（推荐，本地运行）、Docker 容器（容器化部署，HTTP 端点）、Smithery（托管服务，一键搭建）。
*   **关键配置**：必选环境变量 `MEM0_API_KEY`（Mem0 密钥）、`MEM0_DEFAULT_USER_ID`（默认用户 ID）；可选 `MEM0_ENABLE_GRAPH_DEFAULT`（开启图谱记忆）。
*   **典型场景**：将 Mem0 集成至各类 AI 客户端，让 AI 助手（如 Claude、Cursor）拥有自主的记忆管理能力。

### 2. 异步处理模式
*   **核心用途**：提升大篇幅消息、大人数群聊、批量操作的处理性能，避免同步处理的阻塞与超时。
*   **关键参数**：`async_mode=True`（在 `add` 等操作中配置）。
*   **典型场景**：群聊记忆录入、批量导入记忆、大篇幅对话的记忆处理。

### 3. 多环境适配
*   **核心用途**：支持开发、测试、生产等多环境的独立配置与使用，避免环境间数据混淆。
*   **关键配置**：通过不同的 `project_id`/`org_id` 区分不同环境，实现记忆数据的环境隔离。

## 七、Mem0 核心特性总结
1.  **自动化**：上下文自动管理、记忆自动推导、实体自动关联、事件实时通知，减少手动开发成本。
2.  **精细化**：支持多维度自定义配置（指令、条件、阈值、过期时间），适配不同业务的个性化需求。
3.  **场景化**：针对群聊、客服、教育、AI 助手等场景做专属优化，实现记忆的分层 / 分角色 / 分会话管理。
4.  **高兼容性**：通过 MCP 集成支持任意 AI 客户端，无需重复开发，实现记忆能力的通用化复用。
5.  **可优化**：基于反馈机制持续学习，不断提升记忆生成与检索的准确性，适配业务的长期发展。
6.  **高性能**：支持异步处理、批量操作、图谱检索优化，兼顾处理效率与检索精准度。
