# 大模型算法实习生 vs. AI Agent 开发实习生：求职难度与转型分析

这份文档分析了两个岗位的核心区别、求职难度以及你目前转型的可行性。

## 1. 核心区别：造车 vs. 开车

| 维度 | 大模型算法实习生 (Model Training/Algo) | AI Agent 开发实习生 (Model Application) |
| :--- | :--- | :--- |
| **核心工作** | **"造模型"** <br> 预训练 (Pre-training)、微调 (SFT)、RLHF、模型量化、算子优化。 | **"用模型"** <br> 构建 Agent、RAG 系统、Prompt 工程、业务逻辑落地、系统架构。 |
| **日常任务** | 清洗海量数据、写 Deepspeed/Megatron 训练脚本、看 Loss 曲线、读顶会论文 (NeurIPS/ICLR)。 | 写 LangGraph 状态机、调优 RAG 检索效果、对接业务 API、优化 Prompt、做工程化部署。 |
| **技术栈** | PyTorch, Transformers, Deepspeed, CUDA, 线性代数, 概率论。 | Python, LangChain/LangGraph, Vector DB, FastAPI, Docker, 业务逻辑。 |
| **硬件要求** | 动辄 A100/H800 集群 (显存就是命)。 | 消费级显卡 (4090) 或直接调 API 即可。 |

## 2. 找实习的难度对比

**结论：算法岗“卷”的程度远高于开发岗。**

*   **大模型算法岗 (Hard 模式)**:
    *   **门槛极高**: 通常要求 **985/Top 硕博**，最好有 **CCF-A 类顶会论文**，或者有在大厂训练过百亿参数模型的经历。
    *   **HC (Headcount) 少**: 只有头部大厂（阿里、腾讯、字节、百度）和头部创业公司（月之暗面、智谱）养得起算法团队。
    *   **现状**: 神仙打架。

*   **AI Agent 开发岗 (Normal 模式 - 你现在的赛道)**:
    *   **需求量大**: 几乎所有公司（银行、车企、电商、软件公司）都想把 AI 用到自己的业务里，他们不需要造模型，只需要**会用模型解决问题的人**。
    *   **更看重落地**: 面试官更在乎你的 Agent 能不能跑通，能不能解决幻觉，能不能处理复杂逻辑。你手头的 `TongAgent` 项目在这里是**王炸**。

## 3. 你现在转过去的难度大吗？

如果你想从目前的 "Agent 开发" 转到 "纯算法"，难度**比较大**，主要难点在于：

1.  **数学与理论基础**: 需要恶补 Transformer 的底层数学细节（Attention 矩阵计算、位置编码推导等）。
2.  **算力门槛**: 你在学校很难接触到大规模分布式训练环境，简历上这一块会是空白。
3.  **沉没成本**: 你已经把 Agent 这一套玩得很溜了，放弃这个优势去和那些搞了三年算法的人拼论文，性价比不高。

## 4. 推荐策略：做一个“懂算法的 Agent 工程师” (混合型)

与其完全转型，不如在 Agent 基础上**点亮一点微调 (Fine-tuning) 的技能树**。这是性价比最高的路径。

**为什么？**
企业里很多时候 Agent 搞不定（Prompt 调优到头了），就需要对模型进行**轻量级微调 (SFT)**。如果你既会写 Agent，又会微调 Llama 3 / Qwen，那你就是**T0 级别的候选人**。

**如何行动？**
1.  **保持现有优势**: 继续打磨 Agent 项目。
2.  **增加微调亮点**:
    *   学习 **LoRA / QLoRA** 技术（低成本微调）。
    *   找一个开源数据集（比如医疗问答），跑通 **LLaMA-Factory** 或 **Axolotl** 框架。
    *   **实战目标**: 训练一个特定风格的模型（比如“鲁迅风格”的 Qwen），然后把这个模型集成到你的 Agent 里。

## 总结

**不要为了“听起来高大上”去卷纯算法。**
Agent 开发是目前 AI 落地最缺人的方向，也是最容易出成果的方向。你现在的路走得很对，只要把**工程化**（部署、并发、稳定性）和**微调**（SFT）这两个技能补上，你在武汉找实习会非常抢手。
