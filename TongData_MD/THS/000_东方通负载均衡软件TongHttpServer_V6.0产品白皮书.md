1.1  引言 
在当今互联网快速发展的时代，网站和应用系统的性能及可靠性至关重要。
随着用户数量的增长和请求的复杂性增加，服务器的负载和响应时间变得越来越
具有挑战性。在这种背景下，传统的单服务器解决方案往往难以满足现代互联网
应用的需求。因此，企业和开发者需要一种高效、稳定且灵活的解决方案来优化
网络服务的性能，提升用户体验。 
   负载均衡是确保网络服务高可用性和高性能的关键技术之一。它通过将用户
请求分发到多台服务器上，避免单点故障，提升系统的整体处理能力，不仅能够
有效分摊服务器压力，提高系统的响应速度和整体性能，还能提供弹性扩展能力、
优化资源利用、增强安全性和简化维护流程。因此，负载均衡技术已成为现代应
用系统不可或缺的重要组成部分。 
反向代理通过隐藏系统内部结构、缓存内容、SSL 卸载和协议转换等方式提
升系统安全性，同时简化管理和维护。结合负载均衡技术，既可以提高系统的吞
吐率，又增强系统的安全性。 
TongHttpServer V6.0（简称THS）是一款高性能的HTTP 服务器、反向代理/
负载均衡服务器，广泛地应用于电信、金融、政府、交通、能源等各种领域的企
业应用系统中。 
1.2  THS 使用场景

（国际/商密）卸载、邮件代理服务器。

2  产品功能 
THS V6.0 是一款轻量级负载均衡软件，支持 OSI 四层七层负载均衡，具有
占用系统资源少、性能高、支持并发数高、配置简单等优点。常用于配置 HTTP 
静态资源服务器、反向代理服务器、负载均衡服务器、正向代理服务器。 
2.1  Web 服务器常用功能 
2.1.1  静态内容处理 
高效地处理HTML、CSS、JavaScript、图片等静态文件，使用 sendfile 系统
调用直接将文件从磁盘发送到网络，而不需要经过用户空间的数据拷贝，从而减
少了 CPU 资源的消耗和内存复制的开销。 
2.1.2  自动目录索引 
自动目录索引功能（autoindex）允许在 Web 浏览器中查看目录的内容，无
需服务器提供单独的索引页面。在快速浏览目录内容或下载文件等场景非常有
用。 
2.1.3  多虚拟主机 
支持基于域名和基于 IP 的虚拟主机配置，允许在一台服务器上托管多个网
站。 
2.1.4  URL 重写 
支持使用rewrite 规则实现 URL 重写和重定向，支持3xx-5xx 重定向。 
2.1.5  响应优化 
支持gziping、byte ranges、chunked responses 响应内容。 
⚫ gziping：对响应内容进行 gzip 压缩，从而减少传输数据量，提高加载
速度。 
⚫ byte ranges：支持字节范围请求，客户端可以请求部分内容，适用于断点
续传和媒体文件的按需加载。

2.3  负载均衡 
THS 可以高效地分发客户端请求到多台后端服务器，提升系统的性能、可
靠性和扩展性，支持七层HTTP、四层TCP/UDP 协议数据的负载均衡。 
2.3.1  负载均衡算法 
七层支持轮询、哈希（一致性哈希）、IP 哈希、随机、最小连接等负载均
衡算法，四层支持轮询、哈希（一致性哈希）、随机、最小连接负载均衡算法。 
2.3.2  七层请求路由 
根据HTTP 请求的应用层信息（如URL、请求头、HTTP 方法等）将请求转
发到不同的后端服务器，实现灵活的流量管理和请求分发。 
2.3.3  会话一致性 
通过使用Stick session 技术，将同一用户的所有请求始终分配给同一台后端
服务器，以确保会话状态的一致性和连续性，THS 所有负载均衡算法上均支持
Sticky 会话一致性配置。 
2.3.4  DNS 服务发现 
定时解析后端服务器的DNS 记录，动态更新后端服务器列表，从而支持自
动化的负载均衡和故障转移，提升系统的灵活性和可靠性。 
2.3.5  热备 
支持配置后端服务器热备（backup）机制，当主服务器组中的服务器全不可
用时，流量自动切换到备份服务器组。 
2.3.6  长连接 
支持和后端服务器保持长连接，有效提高系统的性能和响应速度，维持长连
接是处理高并发请求和优化资源利用的有效手段。

2.3.7  主动健康检查 
定期向后端服务器发送自定义的健康检查请求，以监测服务器的运行状态。
如果某台服务器没有通过健康检查，THS 会自动将其从负载均衡池中移除，确

2.4  正向代理 
代表客户端向服务器发送请求，并将服务器的响应返回给客户端，正向代理
可以根据 IP 地址、域名或其他规则对访问进行控制和过滤，保护网络安全。THS
支持HTTP、HTTPS 正向代理配置。 
2.5  缓存 
缓存静态内容和动态内容的响应，减少后端服务器的负担，加快客户端的响
应速度。 
2.6  安全 
2.6.1  HTTP 基本认证 
HTTP Basic Authentication 是一种简单的用户认证机制，它通过在 HTTP 请
求中使用用户名和密码来验证客户端的身份。这种认证方式在 HTTP 协议的头
部中传输用户凭证。 
2.6.2  子请求认证 
通过子请求向后端服务器发送认证请求，并根据后端服务器的响应决定是否
允许访问受保护的资源。 
2.6.3  基于 IP 地址的访问控制 
根据客户端的 IP 地址来控制对资源的访问。通过配置访问控制列表，允许
或拒绝特定 IP 地址或 IP 地址范围的请求。
