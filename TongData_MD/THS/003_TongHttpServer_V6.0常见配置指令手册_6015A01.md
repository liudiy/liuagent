主动健康检查用于检测后端工作节点是否正常，对于异常的节点会被标记为 down 状态，待正常后设置为

20.1. health_check
类别
说明

-

说明

A.8. upstream 模块内置变量
字段名
字段说明
$upstream_addr
上游节点的 IP 和端口或 UNIX 域。
• 如果连接了多次使用逗号分隔，如 “192.168.1.1:80, 192.168.1.2:80,
unix:/tmp/sock”。
• 如果未内部重定向 (“X-Accel-Redirect” or error_page) 使用冒号分隔
，如“192.168.1.1:80, 192.168.1.2:80, unix:/tmp/sock :
192.168.10.1:80, 192.168.10.2:80”。
• 如果未选取节点，为 upstream 名字。
$upstream_bytes_received
从上游节点获取数据的大小。如果有多个，与 $upstream_addr 方式一样
进行标识。
$upstream_bytes_sent
发送个上游节点数据大小。如果有多个，与 $upstream_addr 方式一样进
行标识。
$upstream_cache_status
缓存响应状态
，“MISS”、“BYPASS”、“EXPIRED”、“STALE”、“UPDATING”、“REVA
LIDATED” 或者 “HIT”。
$upstream_connect_time
建立连接消耗时间，SSL 包含握手时间。
$upstream_cookie_name
上游节点响应 Set-Cookie 字段中的 name 对应值。
$upstream_header_time
接收上游节点包头消耗时间。
$upstream_http_name
上游节点 http 包头字段。
$upstream_queue_time
请求在队列中保存的时间。若请求无法立即选择 upstream 服务器，则该
请求将被放入队列中。
$upstream_response_length
响应长度。
$upstream_response_time
接收响应消耗的时间。
$upstream_status
上游节点响应码。
