2.4.3. 高可用
高可用模块 httpserverHA 用于防止负载均衡进程发生单点故障，用户使用漂移 IP 访问系统。节点通过定期
执行 bin/monitor.sh 脚本检查本地 httpserver 进程是否异常，并通过 VRRP 协议（linux）、TCP 协议
（windows）将自身信息同步给其他节点，节点优先级使用 priority 判断，priority 高的为主节点，低的为
备节点。
httpserverHA 是 keepalived 可执行文件重命名拷贝，THS 控制台支持主备模式配置，若要配置其他模式如
主主模式、单播等请参考 keepalived 使用说明。
当主节点 httpserver 正常时，漂移 IP 只应出现在主节点上，由主节点对外提供服务。当主节点异常且备节
点 httpserver 正常时，漂移 IP 出现在备份节点上，此时备份节点接管对外提供服务，主节点恢复后漂移 IP 
回到主节点。linux 系统可使用 ip a
 
 命令、windows 系统使用 ipconfig/all  命令查看漂移 IP 是否存在本节
点。

▪节点地址：工作节点 IP 和端口，未指定端口为协议默认端口。http 为 80，https 为 443。
▪权重：(weight) 根据后端节点性能不同进行配置，性能高的可加大权重，默认为 1。
▪最大连接数：（max_conns）到后端工作节点的最大连接数。如果启用了 zone 指令，限制为所有
工作进程的总连接数。
▪最大尝试次数：（max_fails）fail_timeout 时间内与后端节点通信失败的尝试次数，默认为 1。超
出失败尝试次数则认为后端节点在 fail_timeout 内不可用，等待下一个 fail_timeout 再次尝试通
信。
▪周期（秒）：(fail_timeout)，默认为 10 秒。
▪停用：将节点标记为 down 状态，不参与负载均衡。
▪备份节点：(backup)，当其他节点不可用时使用备份节点。
▪DNS 解析：(resolver)，配置后可动态解析域名。

6.15. 页面缓存配置
页面缓存是在包头中设置 Last-Modified、Etag、Expires 字段，是否发生缓存需要浏览器端发送时带上
if_modified_since，If-None-Match 等包头决定。
• Last-Modified
根据文件更新时间来确定是否再次发送内容，浏览器请求时需要携带 If-Modified-Since 包头，THS 默
认开启 Last-Modified 包头输出。
◦如需关闭，在对应 location 中使用 add_header Last-Modified ""
  
 进行关闭。
◦如果禁用 Last-Modified 返回 304 功能，使用 if_modified_since off
 
 进行关闭。
• Etag
给文件资源设置 etag 值，浏览器再次请求时携带 If-None-Match 包头，THS 取出 If-None-Match 和本
地 etag 对比。
如果不同重新发送内容，返回 200 状态码，如果相同，则返回 304。
关闭 Etag 使用 etag 指令, 详情参考 《TongHttpServer_V6.0监控配置手册》。
• Expires
给资源设定过期时间，在有效时间内无需去服务端验证，直接通过浏览器端确认是否过期即可，不会产
生额外的流量。
此方法适合不常变化的资源。
如果资源文件变动较频繁，使用 Expires 来缓存将变得不可靠，另外如果在有效时间内修改了文件，浏
览器需要清除缓存才能加载新的文件。THS 默认关闭了 Expires,可以使用 expires 指令设置。
配置示例，如下所示。
location~.*\.(gif|jpg|jpeg|){
expires 1d;
}
location~.*\.(css|js){
expires 12h;
}

6.20.3. CSRF 防护
攻击者可能在其他网站上构造恶意请求，透导用户在已录目标站点的情况下触发该请求，从而已用户身份执

SSL/TLS 来加密数据包。HTTPS 的主要目的是提供对网站服务器的身份认证，保护
交换数据的隐私与完整性。
SSL
SSL（Secure Sockets Layer，安全套接层）是一种用于保护网络通信安全的加密协
议。它通过在客户端和服务器之间建立安全连接，确保数据在传输过程中不会被窃
取或篡改。
