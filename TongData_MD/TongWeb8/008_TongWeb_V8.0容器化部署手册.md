## 第 8 页

2. 容器化部署 TongWeb
通过 Dockerfile 制作 TongWeb 的 Docker 镜像。
准备工作
• 建议在非容器环境提前启动并配置好 TongWeb 的数据源、通道、应用等资源，以最终在容器内直接使
用。
• 若计划访问容器内 TongWeb 的控制台，可选择提前解除 “信任IP” 的限制，通过管理控制台页面操作或
者修改 console.xml 文件设置 trustedIP 为 “*” 来达到目的。
• 为减小制作镜像的体积，可在制作镜像之前将 ${tongweb.home} 下不需要的版本目录（只保留需要的
一个 version* 目录）和所有版本的 version*.zip 移除，同时清空 temp 和 logs 目录。
• 若需要制作 TongWeb 轻量版本的镜像，需先通过控制台 “版本生成” 模块生成一份轻量版本，再按照上
述和后续步骤进行操作。
操作步骤
1. 在 ${tongweb.home} 同级目录（注：不是在内部）建立 Dockerfile 文件。
如下以 x.x.x.x 版本为例（注：请根据实际版本修改），内容如下所示。
# 注：需更改为合法的基础镜像
FROM openjdk:8u212-jdk-alpine
# 注：按需填写 JVM 启动参数，可选，默认值从 tongweb.xml 中读取
ENV JAVA_OPTS="-Xms512M -Xmx1024M"
# 注：需更改为实际的 TongWeb 文件路径
COPY ./TongWeb /opt/TongWeb
RUN chmod +x /opt/TongWeb/bin/*.sh
CMD ["/bin/sh", "-c", "/opt/TongWeb/bin/standalone.sh"]
2. 在 ${tongweb.home} 同级目录，执行 Docker 镜像构建命令。
docker build -t tongweb:x.x.x.x ./
3. 执行命令，检查构建的镜像。
docker images
4. 执行如下命令，运行 TongWeb 容器。
docker run -p 8088:8088 tongweb:x.x.x.x
注意：
◦此为基础命令，请按需添加 -d --name -it -v -e 等容器支持的参数。
◦若需要访问 TongWeb 的控制台，需添加 -p 9060:9060。

## 第 10 页

3. 最佳实践项
3.1. 基础配置优化
3.1.1. 端口延迟启动（含普罗端口）
目的：
开启 “延迟启动通道” 后，TongWeb 在启动时会优先启动应用，应用启动完成后，再启动端口。开启该功
能可保证当有请求进入系统后所有的应用都是就绪的。
操作：
1. 定位配置节点
在 tongweb.xml 中找到 server  节点（位于 tongweb > server 路径下）。
2. 设置关键参数
delayStartConnector="true"：开启 “延迟启动通道”。
TongWeb 配置（tongweb.xml）：
<server delayStartConnector="true">
3.1.2. 优雅停机（Graceful Shutdown）
目的：
在 TongWeb 服务器停止时，等待正在进行中的客户端请求处理完毕。
操作：
1. 定位配置节点
在 tongweb.xml 中找到 server  节点（位于 tongweb > server 路径下）。
2. 设置关键参数
gracefulStopAwaitMillis="1000"：打开 “优雅停机”，单位：毫秒。设置为 0，表示禁用优雅停机。
TongWeb 配置（tongweb.xml）：
<server gracefulStopAwaitMillis="1000">
3.1.3. 普罗米修斯监控数据上报
目的：
通过总控平台集中监控 TongWeb 运行状态（如 QPS、响应时间、JVM 指标）。

3.1.4. 系统日志与应用日志分离
目的：
将系统打印（System.out、System.err）的日志仅在控制台输出，不写入文件。并将使用
java.util.logging.Logger 打印的应用日志转入到 TongWeb 日志模块统一处理，如存入 TongWeb 的
server.log 日志文件等，从而实现系统日志与应用日志的分离。
操作：
1. 定位配置节点
在 tongweb.xml 中找到 server  节点（位于 tongweb > server > loggers > server 路径下）。
2. 设置关键参数
◦takeoverSystemPrint="true"：接管系统打印（System.out/System.err）。
◦printToConsoleOnly="true"：应用日志仅输出到控制台，不写入文件。
◦takeoverJUL="true"（默认值）：接管 Java Logger（java.util.logging.Logger），日志存入
server.log。
TongWeb 配置（tongweb.xml）：
<server takeoverSystemPrint="true" printToConsoleOnly="true" takeoverJUL="true">

目的：
避免运行时自动扫描部署目录（如 autodeploy ）导致的资源竞争或配置漂移。

## 第 12 页

操作：
1. 定位配置节点
在 tongweb.xml 中找到 server  节点（位于 tongweb > server 路径下）。
2. 设置关键参数
◦autoDeploy="false"：关闭自动部署。

TongWeb 配置（tongweb.xml）：
<server autoDeploy="false" autoDeployDir="autodeploy" startupDeploy="true">
3.1.6. 打开 “镜像模式”
目的：
为提升系统安全性，可选用 “镜像模式” 来运行 TongWeb。当 TongWeb 以镜像模式运行时，会处于 “只
读” 状态，这意味着在此状态下，无法对任何配置参数实施修改操作。
使用说明：
