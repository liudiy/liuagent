进行特定的操作。

1.

3. 
使用Web浏览器；

启动数据库服务器；

在终端窗口中发布命令。

第1章 集群 
1.1 TongWeb 集群 
TongWeb 具备Java EE 的全部功能，支持负载均衡，支持大规模稳定的集群，支持会话（session）的备
份和复制，支持弹性伸缩集群和智能路由，支持性能问题代码级定位的APM 工具，提供集群集中管理工具
等。该版本主要是面向大规模应用环境和对可靠性要求高的应用场景。 
在实际应用中有很多高并发的业务场景，需要构建Web 应用服务器集群予以支撑。TongWeb 集群是由
多个同时运行的JavaEE 应用服务器组成，在外界看来就像一个服务器一样，这多台服务器共同来为客户提
供更高性能的服务。而负载均衡器的任务就是负责多个服务器之间实现合理的请求分配，使这些服务器不
会出现因某一台超负荷、而其他的服务器却没有充分发挥处理能力的情况，即便其中一台服务器宕机，其
它服务器也会继续接管请求进行处理。 
TongWeb 是由东方通自研的TongHttpServer（THS）软件来实现负载均衡功能的，也可以采用硬件负载
设备、开源的 Apache、 nginx、 Haproxy 等配置TongWeb 集群。追求高并发量的情况下采用硬负载，并
发量不高的情况下可采用软负载均衡器THS。 
1.2 TongHttpServer（THS） 
THS 是一款功能强大、稳定高效、高性价比、易于使用、便于维护的负载均衡软件产品。THS 不仅可
以满足用户对负载均衡服务的需求，提升系统可靠性、高效性、可扩展性及资源利用率，还具有很高的性
价比，可以有效降低系统的建设成本、维护成本，并且使用简单、维护便捷。 
THS 的“智能化”特点能够极大地提升系统的运维效率、减少运维工作量、降低因误操作引发事故的
几率。THS 的“云特性”使其既能够为传统架构的业务系统提供具有云计算优势的负载均衡服务，也可以
应用到云计算平台中提供负载均衡服务。提供的主要功能如下： 
⚫ 
支持OSI七层负载均衡功能； 
⚫ 
支持HTTP、HTTPS、AJP等协议； 
⚫ 
支持多种OSI负载均衡常用功能，如会话保持、访问控制等； 
⚫ 
支持多种负载均衡策略； 
⚫ 
支持HA功能、集群功能，支持浮动IP； 
⚫ 
支持TCP、GET、HEAD三种服务器健康检查方式； 
⚫ 
提供安全防护功能，包括访问控制、防御慢查询攻击；

⚫

Session复制是指TongWeb之间session信息可以实现共享，由TDG实现。

⚫

session复制。

3.5.1 验证集群Session 亲和 
验证集群Session 亲和的具体操作步骤如下。 
1. 
在THS上配置session亲和： 
在THS/bin/httpserver.conf的upstream集群中负载策略配置为“ip_hash”或者“sticky”。 
⚫ 
ip_hash负载算法：THS会根据客户端ip进行hash，保证在该节点正常时，相同IP的客户端请求
都转发到同一后端节点。 
⚫ 
sticky负载算法：根据cookie转发。 
如下示例以配置为“ip_pash”为例说明。 
upstream cluster { 
        ip_hash; 
        server 168.1.15.45:8080; 
        server 168.1.15.46:8080; 
        keepalive 32; 
        keepalive_requests 1000; 
        keepalive_timeout 60s; 
    } 
 
注意： 
具体配置方法可参考《TongHttpServer用户手册》的“HTTP负载均衡配置”。 
2. 
创建集群cluster1时会话服务器集群配置里，不勾选“开启session复制”。 
3. 
启动集群cluster1下的TongWeb、负载均衡器、会话服务器，无启动先后顺序要求。

文件集功能是维护管理共享库需要的jar 文件。用户可通过指定一个共享目录，后台将该目录下所有jar 文

共享库的类型分为以下两种： 
⚫

同一个class会加载多次。并且共享库中可访问的静态变量是相互独立的。

第7章 维护常见问题 
7.1 session 亲和与session 复制的关系 
session 亲和（session 保持）是指在集群环境下，同一IE 客户端的请求会被始终分发到同一个TongWeb
节点上。 
session 复制（session 共享）是指TongWeb 之间session 信息可以实现共享。TongWeb 集群可以只配THS 
session 亲和，不配TDG session 复制，这样只会造成客户端请求跳转到另一个TongWeb 节点上时session 信
息丢失。如果TongWeb 集群在THS 配置了session 亲和的同时，开启了TDG session 复制，那么客户端请求
跳转到另一个TongWeb 节点上时session 信息就不丢失，但同时需要维护会话服务器。 
7.2 IE6 快照上传下载问题 
“诊断-快照上传”中使用的快照有严格的文件命名格式：数字4位-2位-2位T2位-2位-2位-3位+4位.zip，
比如2015-12-01T10-22-13.718+0800.zip。如果命名格式不符合上述格式则会提示错误。通常情况下，通过
快照页面下载的文件是严格符合该格式的。但是在IE6 下，其命名会被IE6 添加“[1]”类的符号，比如
2015-12-01T10-22-13[1].718+0800.zip。此时如果需要上传的话，需要手动修改文件名称为符合格式的名称。
比如去掉多余的“[1]”，使用2015-12-01T10-22-13.718+0800.zip 进行上传即可。

错误：

THS 在Linux 上的安装（注：以root 用户登录） 
以安装路径“/home/test/THS”为例，安装过程说明如下： 
1. 
在目标安装目录/home/test/下直接通过命令行进行解压：tar zxvf THS_xxx.tar.gz  
2. 
切换到目标安装目录: cd /home/test/THS/bin  （注：目录不带中文） 
3. 
执行replace.sh脚本: ./replace.sh /home/test/THS 
注：若路径中带空格，则路径参数前后要加英文格式的引号 
注意： 
⚫ 
路径最后面不要带“/”，执行完成后整个目录就不要移动到其它目录了。 
⚫ 
此操作必须对新解压出来的安装包操作，不可以执行一次replace.sh脚本后，整个运行目录移动到
其它路径再次执行。
