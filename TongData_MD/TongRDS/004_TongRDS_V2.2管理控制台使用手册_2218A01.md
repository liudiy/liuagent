第1章 管理控制台简介 
1.1 概述 
TongRDS 管理控制台是基于Java 开发的应用程序，它为TongRDS 提供了服务/节点的配置管理、部署、
升级、监控、数据维护等功能，支持单点模式、哨兵模式、集群模式、可伸缩集群模式四种部署模式。 
1.2 关键概念 
由于后续对于此平台的功能描述中涉及诸多基本概念，本节将介绍涉及的术语和相关名词定义。 
⚫ 
管理控制台：即TongRDS 管理控制台，支持通过WEB 的方式进行TongRDS 服务以及节点的管
理、TongRDS 服务以及节点的监控、TongRDS 服务的数据操作等功能。 
⚫ 
节点：即TongRDS 的节点，是部署运行的一个实例，也是部署运行的最小独立单元。分为服务
节点、哨兵节点两种类型。 
⚫ 
配置模板：是TongRDS 的配置文件，分为服务节点配置模板和哨兵节点配置模板两种类型，服
务节点配置模板对应TongRDS 中的pmemdb/etc/cfg.xml 配置文件，哨兵节点配置模板对应
TongRDS 中的pmemdb/etc/sentinel.xml 配置文件。为了简化配置，系统会提供默认的配置模板。 
⚫ 
节点管理器：节点管理器负责节点的部署、启动、停止、重启、卸载、更改配置等操作；在添加
主机时，如果可以通过Linux/Unix 系统下的SSH 服务进行登录，在主机添加成功后会自动完成节
点管理器的安装；如果没有可用的SSH 服务，需要系统管理员手动进行安装和并控制进行连接。 
⚫ 
服务：即RDS 服务，服务支持单点模式、哨兵模式、集群模式、代理集群模式四种部署模式；单
点部署模式下，仅可添加一个服务节点；哨兵部署模式下，最少需要添加一个服务节点和一个哨
兵节点；集群部署模式下，最少需要添加一个分片，每个分片最少需要添加两个服务节点；代理
集群模式下，最少有一个代理节点，最少需要添加一个分片，每个分片最少需要添加两个服务节
点。

8.2 数据恢复 
系统数据恢复功能，主要用于因软硬件环境异常，导致的数据文件损坏后的恢复。 
注意该功能操作前必须先停止正在运行的控制台程序，数据恢复是导入备份时点的全量数据，当前数
据将被完全覆盖且无法恢复。 
操作步骤如下： 
1. 
停止管理控制台： 
./console.sh stop 
2. 
在APPHOME/data/dbbak 目录下找到需要导入恢复的文件名，如：export-202211150130.zip 
3. 
进行恢复操作：进入APPHOME 目录，执行如下命令： 
java -jar lib/console-admin.jar --restore export-202211150130.zip 
4. 
恢复前系统会提示注意事项，选择Y 开始备份恢复操作。
